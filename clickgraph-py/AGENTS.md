# clickgraph-py — Agent Guide

> **Purpose**: Python bindings for ClickGraph's embedded mode using UniFFI.
> Provides a Pythonic API over the auto-generated UniFFI ctypes bridge,
> sharing the same `clickgraph-ffi` crate used by the Go bindings.

## Architecture

```
Python application code
  └── clickgraph/__init__.py (Pythonic wrapper: Database, Connection, QueryResult)
        └── clickgraph/_ffi.py (auto-generated UniFFI ctypes bridge)
              └── libclickgraph_ffi.so (shared library)
                    └── clickgraph-ffi (Rust) → clickgraph-embedded → chdb
```

## File Overview

```
clickgraph/
├── __init__.py    (470+ lines) ← Public API: Database, Connection, QueryResult, GraphResult, StoreStats
├── _ffi.py        (2346 lines) ← Auto-generated by uniffi-bindgen (NEVER EDIT)
├── libclickgraph_ffi.so        ← Dev symlink to build output (gitignored)
tests/
├── test_bindings.py  (401 lines)  ← 40 sql_only tests (no chdb required)
├── test_chdb_e2e.py  (856 lines)  ← 72 Parquet-based e2e tests
pyproject.toml                      ← Package configuration
README.md                          ← Documentation with usage examples
.gitignore                         ← Excludes .so symlinks and bytecode
```

## Key API

```python
import clickgraph

# Open database with schema
db = clickgraph.Database("schema.yaml")
conn = db.connect()

# Execute Cypher query — dict-style rows
for row in conn.query("MATCH (u:User) RETURN u.name"):
    print(row["u.name"])

# Kuzu-compatible cursor style
result = conn.execute("MATCH (u:User) RETURN u.name")
while result.has_next():
    values = result.get_next()  # list of values
    print(values[0])

# SQL-only mode (no chdb needed)
db = clickgraph.Database.sql_only("schema.yaml")
sql = db.connect().query_to_sql("MATCH (u:User) RETURN u.name")

# Export to file
conn.export("MATCH ...", "users.parquet")
conn.export("MATCH ...", "users.csv", format="csv")

# S3 credentials
db = clickgraph.Database("schema.yaml",
    s3_access_key_id="AKIA...",
    s3_secret_access_key="...",
    s3_region="us-east-1")

# Hybrid remote query + local storage
db = clickgraph.Database("schema.yaml",
    session_dir="/tmp/local-graphrag",
    remote_url="http://ch-cluster:8123",
    remote_user="analyst",
    remote_password="secret",
    remote_database="analytics")
conn = db.connect()

graph = conn.query_remote_graph(
    "MATCH (u:User)-[r:FOLLOWS]->(f:User) RETURN u, r, f LIMIT 10000")
stats = conn.store_subgraph(graph)
# stats.nodes_stored, stats.edges_stored

# Structured graph result (local)
graph = conn.query_graph("MATCH (u:User) RETURN u LIMIT 10")
# graph.nodes → list of dicts with id, labels, properties
# graph.edges → list of dicts with id, type_name, from_id, to_id, properties
```

## Key Design Decisions

### Thin Wrapper over Auto-Generated Code
`__init__.py` is a thin Pythonic layer (~379 lines) that:
- Converts UniFFI `Value` enum variants to native Python types
- Converts UniFFI `Row` objects to Python dicts
- Adds `__iter__`, `__len__`, `__getitem__` for ergonomic access
- Provides `execute()` and `run()` aliases for compatibility
- Normalizes export format names

### _ffi.py Is Auto-Generated
**Never edit `_ffi.py` manually.** Regenerate with:
```bash
pip install uniffi-bindgen==0.29.5  # must match Cargo.toml uniffi version
uniffi-bindgen generate --library target/debug/libclickgraph_ffi.so \
  --language python -o clickgraph-py/clickgraph/
```

### Value Conversion
UniFFI generates `Value` as an enum with nested dataclasses:
```python
Value.NULL()              → None
Value.STRING(v="Alice")   → "Alice"
Value.INT64(v=42)         → 42
Value.FLOAT64(v=3.14)     → 3.14
Value.BOOL(v=True)        → True
Value.LIST(items=[...])   → [...]
Value.MAP(entries=[...])  → {...}
```
The `_value_to_python()` function handles this conversion.

### Export Format Aliases
The `_parse_format()` function normalizes user-facing names to FFI-accepted strings:
- `csv`, `csvwithnames` → `"csv"` (FFI does the ClickHouse format mapping)
- `ndjson`, `jsonl`, `jsoneachrow` → `"ndjson"`
- `parquet`, `pq` → `"parquet"`

Format parsing happens in **both** the Python wrapper (user-friendly aliases) and
the Rust FFI layer (canonical format → ClickHouse format). Don't duplicate the
ClickHouse format names in Python.

## Conventions

- **API parity**: The public API must be identical to the PyO3 version (`clickgraph-py`)
- **Test gating**: chdb e2e tests require `CLICKGRAPH_CHDB_TESTS=1` env var
- **Parquet fixtures**: e2e tests use PyArrow to create Parquet files (not CSV)
- **Schema source URIs**: e2e tests use bare file paths (`/path/file.parquet`),
  which the source resolver auto-detects as Parquet format

## Dev Setup

```bash
# Build the shared library
cargo build -p clickgraph-ffi

# Create dev symlink (gitignored)
ln -sf /path/to/target/debug/libclickgraph_ffi.so clickgraph/libclickgraph_ffi.so

# Find chdb library path
CHDB_DIR=$(dirname $(find target/debug/build -name "libchdb.so" -path "*/chdb-rust-*/out/*"))

# Run sql_only tests
PYTHONPATH=. python3 -m pytest tests/test_bindings.py -v

# Run all tests including chdb e2e
LD_LIBRARY_PATH="$CHDB_DIR" CLICKGRAPH_CHDB_TESTS=1 PYTHONPATH=. \
  python3 -m pytest tests/ -v
```
